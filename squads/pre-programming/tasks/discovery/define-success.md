# Task: Definir Critérios de Sucesso

## Objetivo
Estabelecer critérios claros, mensuráveis e acordados que definem o sucesso do projeto, criando uma referência objetiva para avaliar se o projeto atingiu seus objetivos.

## Input Necessário
- Project Brief com objetivos do projeto.
- Discovery com declaração do problema.
- Expectativas dos stakeholders mapeadas.
- Métricas do cenário atual (baseline).

## Agentes Envolvidos
- **Agente de Discovery:** Facilita a definição de critérios.
- **Stakeholders:** Definem o que consideram sucesso.
- **Agente de Qualidade:** Garante que os critérios são mensuráveis e testáveis.

## Passos

### 1. Coletar Visões de Sucesso
- Perguntar a cada stakeholder principal: "Como você saberá que este projeto foi bem-sucedido?"
- Documentar respostas exatas, sem interpretar.

### 2. Categorizar Critérios
- **Negócio:** Métricas de impacto no negócio (receita, custo, conversão).
- **Usuário:** Métricas de experiência (NPS, tempo de tarefa, adoção).
- **Técnicos:** Métricas de qualidade técnica (performance, disponibilidade, bugs).
- **Processo:** Métricas de eficiência operacional (lead time, automação).

### 3. Tornar Critérios Mensuráveis
- Transformar cada critério em formato SMART:
  - Específico: o que exatamente será medido?
  - Mensurável: qual a métrica e ferramenta?
  - Atingível: é realista dado o escopo?
  - Relevante: está alinhado ao objetivo?
  - Temporal: até quando deve ser atingido?

### 4. Definir Baseline e Meta
- Para cada critério, documentar:
  - Valor atual (baseline).
  - Valor alvo (meta).
  - Valor mínimo aceitável (piso).
  - Método de medição.

### 5. Validar com Stakeholders
- Apresentar os critérios formatados.
- Confirmar que todos concordam com metas e prazos.
- Registrar acordo formal.

## Output Esperado
- Lista de critérios de sucesso mensuráveis e acordados.
- Baseline documentado para cada métrica.
- Método de medição definido para cada critério.
- Acordo formal dos stakeholders.

## Checklist de Validação
- [ ] Cada critério é mensurável e tem um valor alvo claro.
- [ ] Há baseline (valor atual) documentado para cada métrica.
- [ ] Os critérios cobrem perspectivas de negócio, usuário e técnica.
- [ ] As metas são realistas dado o escopo e restrições.
- [ ] O método de medição está definido (ferramenta, frequência).
- [ ] Stakeholders revisaram e concordaram com os critérios.
- [ ] Há prazo definido para avaliação de cada critério.
