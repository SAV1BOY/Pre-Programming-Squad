# Discovery Completeness Checklist

## Propósito
Garantir que a fase de discovery cobriu todos os aspectos necessários antes de avançar para escopo.

## Quando Usar
- Durante a fase correspondente do pipeline de pré-programação
- Em revisões de qualidade antes de avançar para a próxima fase
- Quando o Readiness Gatekeeper solicita verificação

---

## Checklist

### Entendimento do Problema
- [ ] Problema raiz foi identificado (não apenas sintoma)
- [ ] Contexto de negócio está documentado
- [ ] Usuários/atores afetados estão mapeados
- [ ] Impacto quantificado está registrado
- [ ] Perguntas abertas foram respondidas ou escaladas

### Completude
- [ ] Todas as fontes de informação foram consultadas
- [ ] Stakeholders relevantes foram entrevistados
- [ ] Dados históricos foram analisados
- [ ] Soluções existentes foram avaliadas
- [ ] Assumptions estão explícitas e marcadas para validação

---

## Critérios de Aprovação
- **Mínimo:** 80% dos itens marcados
- **Recomendado:** 100% dos itens marcados
- **Bloqueador:** Qualquer item crítico não atendido deve ser escalado

## Red Flags
- Mais de 3 itens não marcados sem justificativa
- Itens marcados sem evidência de verificação real
- Checklist preenchido em menos de 5 minutos (superficialidade)
