# KPI-Driven Planning Framework

## Propósito
Framework para orientar planejamento de pré-programação com base em KPIs mensuráveis de qualidade, eficiência e valor entregue.

## Quando Usar
- Durante a fase de pré-programação quando o contexto exige este tipo de análise
- Quando há complexidade ou incerteza no aspecto coberto por este framework
- Como parte do pipeline padrão para projetos de média/alta complexidade

## Processo / Passos

### Passo 1
Selecionar KPIs relevantes para o tipo de projeto

### Passo 2
Definir baselines e targets para cada KPI

### Passo 3
Mapear atividades do pipeline que impactam cada KPI

### Passo 4
Instrumentar coleta de dados durante o processo

### Passo 5
Revisar KPIs em cada gate transition

### Passo 6
Alimentar RalphLoop com dados de KPI para melhoria contínua

## Armadilhas Comuns

- **Medir tudo sem foco — excesso de métricas dilui atenção**
- **Usar KPIs como punição em vez de aprendizado**
- **Não atualizar targets com base em dados históricos**
- **Goodhart's Law: quando a métrica vira meta, deixa de ser boa métrica**

## Output Esperado
Documento estruturado com análise, decisões e justificativas seguindo os passos acima.
