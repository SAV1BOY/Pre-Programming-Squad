# Avaliar Build vs Buy

## Descrição
Avaliar se componentes do projeto devem ser construídos internamente ou adquiridos/reutilizados.

## Agentes Envolvidos
Consultar `config.yaml` para roteamento de agentes por tipo de task.

## Passos

### 1. Identificar componentes candidatos a buy/reuse

### 2. Avaliar opções disponíveis no mercado

### 3. Comparar custo total de ownership (build vs buy)

### 4. Avaliar riscos de lock-in e manutenção

### 5. Documentar decisão no ADR

## Artefatos de Saída
- Documento de resultado seguindo template correspondente
- Atualização dos registries relevantes
- Input para a próxima fase do pipeline
