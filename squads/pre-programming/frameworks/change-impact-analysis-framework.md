# Change Impact Analysis Framework

## Propósito
Framework para avaliar o impacto de mudanças propostas em sistemas existentes, equipes e processos.

## Quando Usar
- Durante a fase de pré-programação quando o contexto exige este tipo de análise
- Quando há complexidade ou incerteza no aspecto coberto por este framework
- Como parte do pipeline padrão para projetos de média/alta complexidade

## Processo / Passos

### Passo 1
Definir a mudança proposta em termos claros

### Passo 2
Mapear todos os sistemas e componentes afetados

### Passo 3
Identificar equipes e stakeholders impactados

### Passo 4
Avaliar riscos de cada impacto (probabilidade × severidade)

### Passo 5
Definir estratégia de mitigação para impactos críticos

### Passo 6
Planejar comunicação e rollout

## Armadilhas Comuns

- **Subestimar efeitos em cascata de mudanças**
- **Não considerar impactos organizacionais além dos técnicos**
- **Avaliar impacto apenas no cenário ideal**
- **Não consultar equipes afetadas antes de planejar**

## Output Esperado
Documento estruturado com análise, decisões e justificativas seguindo os passos acima.
