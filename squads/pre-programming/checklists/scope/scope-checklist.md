# Scope Definition Audit Checklist

## Propósito
Garantir que o escopo foi definido com clareza, incluindo o que está dentro, fora e o que foi cortado com justificativa.

## Quando Usar
- Durante a fase correspondente do pipeline de pré-programação
- Em revisões de qualidade antes de avançar para a próxima fase
- Quando o Readiness Gatekeeper solicita verificação

---

## Checklist

### Definição de Escopo
- [ ] In-scope está explicitamente documentado
- [ ] Out-of-scope está explicitamente documentado
- [ ] Cortes de escopo têm justificativa registrada
- [ ] MVP está definido com critérios claros
- [ ] Fases futuras estão sinalizadas mas não comprometidas

### Priorização
- [ ] Itens estão priorizados por valor vs esforço
- [ ] Dependências entre itens estão mapeadas
- [ ] Quick wins estão identificados
- [ ] Bloqueadores estão sinalizados
- [ ] Critérios de repriorização estão definidos

---

## Critérios de Aprovação
- **Mínimo:** 80% dos itens marcados
- **Recomendado:** 100% dos itens marcados
- **Bloqueador:** Qualquer item crítico não atendido deve ser escalado

## Red Flags
- Mais de 3 itens não marcados sem justificativa
- Itens marcados sem evidência de verificação real
- Checklist preenchido em menos de 5 minutos (superficialidade)
