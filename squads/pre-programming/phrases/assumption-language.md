# Frases: Linguagem de Premissas

## Contexto
Usar quando decisões estão sendo tomadas com base em premissas que precisam ser explicitadas, registradas e validadas. Premissas não documentadas são a principal fonte de surpresas em projetos.

## Frases-Modelo

1. "Essa decisão assume que [premissa]. Vamos registrar isso e definir quando e como validar."

2. "Estamos assumindo que a equipe do parceiro entrega até março. E se não entregar? Qual é o plano B?"

3. "Toda premissa é uma aposta. Vamos garantir que sabemos quais apostas estamos fazendo e quanto perdemos se errarmos."

4. "Isso é fato confirmado ou premissa? Se é premissa, precisa ir para o log de premissas."

5. "As 3 premissas mais arriscadas deste projeto são: [1], [2] e [3]. Vamos priorizar a validação delas."

6. "Premissa validada vira fato. Premissa invalidada vira replanejamento. Premissa ignorada vira crise."

7. "Se essa premissa estiver errada, o impacto é de [X semanas/R$Y]. Vale investir [tempo] para validá-la agora."

8. "Vamos fazer um exercício: quais das nossas premissas, se estiverem erradas, inviabilizam o projeto inteiro?"

## Quando Usar
- Quando decisões dependem de informações não confirmadas.
- Quando a equipe está tratando suposições como verdades.
- Em revisões de planejamento para auditar a base das decisões.

## Quando NÃO Usar
- Quando a informação já foi confirmada com a fonte primária.
- Para criar paralisia questionando tudo o tempo todo.
- Em contextos onde todas as premissas já estão documentadas e gerenciadas.

## Tom Recomendado
Metódico e preventivo. Não se trata de desconfiar de tudo, mas de ter disciplina em separar o que sabemos do que achamos. Toda premissa registrada é uma premissa gerenciada.
