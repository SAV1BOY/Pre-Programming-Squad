# Task: Mapear Stakeholders

## Objetivo
Identificar todas as partes interessadas no projeto, mapear suas influências, interesses e expectativas, e definir estratégia de engajamento para garantir alinhamento contínuo.

## Input Necessário
- Project Brief com stakeholders iniciais.
- Organograma ou estrutura organizacional.
- Contexto sobre áreas afetadas pelo projeto.

## Agentes Envolvidos
- **Agente de Alinhamento:** Conduz o mapeamento e define estratégias.
- **Agente de Intake:** Fornece contexto sobre o solicitante.
- **Stakeholder principal:** Valida o mapa e indica outros stakeholders.

## Passos

### 1. Identificação Inicial
- Listar todos os stakeholders mencionados no brief.
- Expandir a lista perguntando: "Quem mais é afetado? Quem pode bloquear? Quem precisa aprovar?"
- Incluir stakeholders indiretos (compliance, segurança, infra, suporte).

### 2. Classificar Cada Stakeholder
- Para cada stakeholder, avaliar:
  - Nível de influência (poder de decisão ou bloqueio).
  - Nível de interesse (quanto se importa com o resultado).
  - Posição (apoiador, neutro, resistente).

### 3. Montar Matriz Poder x Interesse
- Posicionar stakeholders na matriz.
- Definir estratégia por quadrante:
  - Alta influência + Alto interesse → Gerenciar de perto
  - Alta influência + Baixo interesse → Manter satisfeito
  - Baixa influência + Alto interesse → Manter informado
  - Baixa influência + Baixo interesse → Monitorar

### 4. Mapear Expectativas e Conflitos
- Documentar a expectativa principal de cada stakeholder.
- Identificar conflitos entre expectativas.
- Propor resolução para conflitos encontrados.

### 5. Definir Plano de Comunicação
- Para cada grupo de stakeholders, definir canal, frequência e conteúdo.
- Designar responsáveis pela comunicação.

### 6. Documentar
- Preencher o template `stakeholder-alignment-template.md`.

## Output Esperado
- Mapa de stakeholders completo com classificação.
- Matriz Poder x Interesse preenchida.
- Plano de comunicação definido.
- Conflitos de expectativa identificados com proposta de resolução.

## Checklist de Validação
- [ ] Todos os stakeholders relevantes estão identificados (incluindo indiretos).
- [ ] Cada stakeholder tem influência e interesse avaliados.
- [ ] A matriz Poder x Interesse está preenchida.
- [ ] Conflitos de expectativa estão documentados.
- [ ] Plano de comunicação está definido com responsáveis.
- [ ] O stakeholder principal validou o mapa.
- [ ] Stakeholders de compliance e segurança foram considerados.
