# Frases: Linguagem de Dependências

## Contexto
Usar quando é necessário comunicar dependências entre equipes, sistemas ou decisões, garantindo que bloqueios e riscos sejam visíveis e gerenciados proativamente.

## Frases-Modelo

1. "Temos uma dependência da equipe X para fornecer a API até [data]. Se atrasar, nosso sprint Y fica bloqueado. Vamos alinhar agora."

2. "Essa funcionalidade depende de uma decisão que ainda não foi tomada. Quem decide e até quando?"

3. "Dependência identificada não é dependência gerenciada. Precisamos de um responsável, um prazo e um plano B."

4. "Toda dependência externa é um risco. Qual é o nosso plano se a equipe X não entregar no prazo?"

5. "Podemos desacoplar essa dependência de alguma forma? Trabalhar com mock enquanto a integração real não está pronta?"

6. "A cadeia de dependências atual tem 4 elos. Se qualquer um atrasar, a entrega inteira atrasa. Vamos identificar caminhos paralelos."

7. "Dependências técnicas são gerenciáveis. Dependências de decisão são mais perigosas — ninguém bloqueia um pipeline, mas decisões pendentes bloqueiam equipes inteiras."

## Quando Usar
- Quando dependências entre equipes ou sistemas são identificadas.
- Quando há risco de bloqueio por dependência não gerenciada.
- Para promover desacoplamento e trabalho paralelo.

## Quando NÃO Usar
- Para criar alarme sobre dependências triviais já gerenciadas.
- Para transferir responsabilidade ("é culpa da outra equipe").
- Quando a dependência é inevitável e já está no plano.

## Tom Recomendado
Proativo e colaborativo. O objetivo é tornar dependências visíveis e gerenciáveis, não culpar outras equipes. Sempre proponha soluções (mock, desacoplamento, alinhamento antecipado) junto com a identificação do problema.
