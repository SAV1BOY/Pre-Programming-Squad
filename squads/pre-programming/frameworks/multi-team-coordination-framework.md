# Multi-Team Coordination Framework

## Propósito
Framework para coordenar trabalho entre múltiplos times quando um projeto afeta mais de um squad ou domínio.

## Quando Usar
- Durante a fase de pré-programação quando o contexto exige este tipo de análise
- Quando há complexidade ou incerteza no aspecto coberto por este framework
- Como parte do pipeline padrão para projetos de média/alta complexidade

## Processo / Passos

### Passo 1
Mapear todos os times envolvidos e suas responsabilidades

### Passo 2
Definir pontos de contato e canais de comunicação

### Passo 3
Estabelecer cadência de sincronização

### Passo 4
Definir artefatos compartilhados e ownership

### Passo 5
Mapear dependências entre entregas de cada time

### Passo 6
Planejar resolução de conflitos e escalação

## Armadilhas Comuns

- **Assumir que todos os times têm o mesmo contexto**
- **Não definir ownership claro de artefatos compartilhados**
- **Sincronizações muito frequentes ou muito raras**
- **Não ter processo de resolução de conflitos**

## Output Esperado
Documento estruturado com análise, decisões e justificativas seguindo os passos acima.
