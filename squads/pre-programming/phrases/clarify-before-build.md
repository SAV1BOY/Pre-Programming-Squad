# Frases: Clarificar Antes de Construir

## Contexto
Usar quando a equipe está tentando avançar para implementação sem ter clareza suficiente sobre o problema, os requisitos ou o escopo. Estas frases reforçam a disciplina de entender antes de fazer.

## Frases-Modelo

1. "Antes de escrever a primeira linha de código, precisamos ter certeza de que estamos resolvendo o problema certo."

2. "Qual é exatamente o problema que estamos tentando resolver? Vamos separar o problema da solução."

3. "Ainda temos perguntas sem resposta que podem mudar o escopo. Vamos resolvê-las antes de comprometer a equipe."

4. "Investir 2 dias em clarificação agora pode economizar 2 semanas de retrabalho depois."

5. "Não estamos atrasando o projeto ao clarificar — estamos evitando construir a coisa errada."

6. "Se não conseguimos explicar o que vamos construir em 2 frases, não entendemos o suficiente para começar."

7. "Cada ambiguidade não resolvida agora vira uma decisão que o desenvolvedor vai tomar sozinho — e pode ser a decisão errada."

8. "Vamos confirmar com o stakeholder antes de assumir. Suposições baratas na fase de planejamento viram retrabalho caro na implementação."

## Quando Usar
- Quando a equipe quer pular a fase de discovery ou clarificação.
- Quando há pressão para "começar logo" sem entendimento claro.
- Quando requisitos estão vagos mas a equipe está avançando mesmo assim.

## Quando NÃO Usar
- Quando o problema já está bem entendido e documentado.
- Quando o projeto é uma tarefa simples e bem definida.
- Para bloquear o progresso sem oferecer um caminho para avançar.

## Tom Recomendado
Firme mas construtivo. Não é sobre impedir o progresso, é sobre garantir que o progresso seja na direção certa. Sempre acompanhe a frase com uma proposta de próximo passo concreto.
