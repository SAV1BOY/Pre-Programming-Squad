# Task: Atualizar Checklists

## Objetivo
Incorporar lições aprendidas nos checklists de validação existentes, garantindo que erros passados não se repitam em projetos futuros.

## Input Necessário
- Relatório de análise de falhas de planejamento.
- Lições aprendidas documentadas.
- Checklists de validação atuais (de todos os templates e tasks).

## Agentes Envolvidos
- **Agente de Aprendizado:** Propõe atualizações nos checklists.
- **Agente de Qualidade:** Valida que as atualizações são relevantes e práticas.
- **Equipe de Pré-programação:** Revisa e aprova mudanças.

## Passos

### 1. Mapear Lições para Checklists
- Para cada lição aprendida, identificar qual checklist deveria cobri-la.
- Verificar se já existe um item similar no checklist.
- Se existe, avaliar se precisa ser reforçado ou detalhado.
- Se não existe, propor novo item.

### 2. Redigir Novos Itens
- Cada item de checklist deve ser:
  - Verificável (Sim/Não, não subjetivo).
  - Específico (não genérico demais).
  - Acionável (indica claramente o que fazer).

### 3. Revisar Itens Existentes
- Remover itens que nunca identificaram problemas (ruído).
- Consolidar itens redundantes.
- Reordenar por prioridade/importância.

### 4. Validar com a Equipe
- Apresentar mudanças propostas.
- Coletar feedback sobre praticidade.
- Ajustar com base no feedback.

### 5. Implementar Mudanças
- Atualizar os templates e documentos de task.
- Versionar as mudanças.
- Comunicar as atualizações à equipe.

## Output Esperado
- Checklists atualizados com lições aprendidas incorporadas.
- Itens obsoletos removidos.
- Comunicação sobre mudanças enviada à equipe.

## Checklist de Validação
- [ ] Todas as lições aprendidas relevantes foram mapeadas a checklists.
- [ ] Novos itens são verificáveis e específicos.
- [ ] Itens obsoletos ou redundantes foram removidos.
- [ ] A equipe revisou e aprovou as mudanças.
- [ ] Templates e documentos foram atualizados.
- [ ] Mudanças foram comunicadas.
- [ ] Versão dos documentos foi incrementada.
