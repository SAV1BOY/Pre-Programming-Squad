# Frases: Linguagem de Prontidão

## Contexto
Usar quando é necessário avaliar e comunicar o nível de prontidão do projeto para avançar de fase, garantindo que "pronto" signifique genuinamente pronto e não "vamos torcer para dar certo".

## Frases-Modelo

1. "Pronto significa que a equipe de dev pode começar a sprint 1 sem precisar nos perguntar coisas básicas. Estamos nesse ponto?"

2. "Readiness score de 65/100 significa que estamos 65% preparados — e 35% descobrindo as coisas durante o desenvolvimento."

3. "O que falta para estarmos prontos? Vamos listar e definir quem resolve cada item."

4. "Pronto com ressalvas é aceitável. Pronto com itens bloqueantes não resolvidos não é pronto."

5. "A equipe de dev é o melhor juiz de prontidão. Se eles dizem que falta algo, falta algo."

6. "Cada item pendente que empurramos para o dev resolver durante a implementação é dívida de planejamento."

7. "Estar pronto não é ter todos os detalhes. É ter clareza suficiente para que a equipe tome boas decisões nos detalhes."

8. "O custo de começar sem estar pronto é sempre maior que o custo de esperar mais alguns dias para resolver pendências."

9. "Readiness é um espectro, não um binário. A pergunta não é 'estamos prontos?', mas 'estamos prontos o suficiente para o risco ser aceitável?'."

## Quando Usar
- Ao avaliar se o projeto pode avançar para a próxima fase.
- Quando há pressão para começar o desenvolvimento antes da prontidão.
- Em revisões de readiness com a equipe.

## Quando NÃO Usar
- Para bloquear o progresso com perfeccionismo ("nunca está pronto o suficiente").
- Em projetos exploratórios onde descobrir durante a implementação é esperado.
- Quando a equipe já demonstrou maturidade para lidar com lacunas.

## Tom Recomendado
Realista e construtivo. Prontidão não é sobre perfeição — é sobre ter o mínimo necessário para que a equipe trabalhe com eficácia. Seja honesto sobre o que falta, mas também reconheça o que está pronto.
