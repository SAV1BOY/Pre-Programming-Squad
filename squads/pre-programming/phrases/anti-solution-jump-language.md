# Frases: Anti-Salto para Solução

## Contexto
Usar quando alguém (stakeholder, dev, ou até o próprio time de planejamento) pula diretamente para uma solução técnica sem ter explorado o problema adequadamente. O "salto para solução" é um dos vieses mais comuns em engenharia.

## Frases-Modelo

1. "Essa é uma solução interessante. Mas qual é exatamente o problema que ela resolve?"

2. "Vamos dar um passo atrás. Antes de discutir 'como', precisamos alinhar o 'o quê' e o 'por quê'."

3. "Quando o stakeholder pede 'quero um app', a pergunta real é: qual problema o app resolve que outra solução não resolveria?"

4. "Estamos casados com essa tecnologia ou abertos a avaliar opções? Às vezes a solução mais simples não é a mais óbvia."

5. "A solução já estava na cabeça antes do problema ser entendido. Vamos garantir que estamos resolvendo a causa, não tratando o sintoma."

6. "Toda vez que começamos com a solução e voltamos para o problema, corremos o risco de definir o problema para justificar a solução."

7. "Entendo a sugestão de usar [tecnologia X]. Mas primeiro: quais são os critérios que a solução precisa atender? Depois avaliamos qual tecnologia atende melhor."

8. "Problema bem definido é metade da solução. Problema mal definido é 100% do retrabalho."

## Quando Usar
- Quando stakeholders dizem "quero um [solução]" ao invés de descrever o problema.
- Quando desenvolvedores propõem tecnologia antes de entender o contexto.
- Quando a discussão de arquitetura começa antes da discovery terminar.

## Quando NÃO Usar
- Quando o problema já está bem entendido e a discussão de soluções é o próximo passo natural.
- Quando a restrição tecnológica é real (obrigatório usar a stack X).
- Para bloquear sugestões legítimas que vêm de experiência prática.

## Tom Recomendado
Curioso e gentil. Não é sobre invalidar a sugestão — é sobre garantir que a sugestão seja avaliada no contexto certo. "Ótima ideia, vamos guardá-la para quando tivermos o problema definido" é melhor que "Pare de falar de solução".
