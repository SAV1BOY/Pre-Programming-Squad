# Task: Executar Go/No-Go

## Objetivo
Conduzir a reunião formal de Go/No-Go para decidir se o projeto está pronto para avançar para a fase de desenvolvimento, com base em critérios objetivos e consenso dos decisores.

## Input Necessário
- Score de readiness calculado.
- Pacote de desenvolvimento montado.
- Lista de itens pendentes e bloqueantes.
- Participação de todos os decisores.

## Agentes Envolvidos
- **Agente de Readiness:** Facilita a reunião.
- **Decisor Final:** Toma a decisão (geralmente VP/Diretor).
- **Tech Lead:** Representa a perspectiva técnica.
- **Product Owner:** Representa a perspectiva de negócio.
- **Stakeholders-chave:** Participam do voto.

## Passos

### 1. Preparar a Reunião
- Enviar antecipadamente: score de readiness, lista de critérios, itens pendentes.
- Confirmar participação de todos os decisores.
- Preparar apresentação concisa (15 min máx).

### 2. Apresentar o Status
- Apresentar score de readiness por dimensão.
- Destacar critérios verdes, amarelos e vermelhos.
- Apresentar itens pendentes e plano para resolvê-los.

### 3. Discutir Riscos e Itens Pendentes
- Para cada item vermelho, discutir:
  - É realmente bloqueante?
  - Pode ser resolvido em paralelo ao início do dev?
  - Qual o risco de prosseguir sem resolvê-lo?

### 4. Coletar Votos
- Cada decisor vota: Go, No-Go ou Go com condições.
- Cada voto deve ter justificativa registrada.
- Mesmo votos dissidentes são registrados (discordância saudável).

### 5. Registrar Decisão
- Preencher o template `go-no-go-template.md`.
- Se Go com condições: definir condições, prazos e consequências.
- Se No-Go: definir o que precisa mudar e quando reavaliar.

### 6. Comunicar Resultado
- Informar toda a organização sobre a decisão.
- Se Go: confirmar data de início do desenvolvimento.
- Se No-Go: comunicar com transparência o motivo e o plano.

## Output Esperado
- Decisão formal registrada (Go / No-Go / Go com condições).
- Justificativas e votos documentados.
- Se Go: data de início do dev confirmada.
- Se No-Go: plano de ação e data de reavaliação.

## Checklist de Validação
- [ ] Todos os decisores participaram ou delegaram formalmente.
- [ ] Critérios de Go foram avaliados com evidências.
- [ ] Votos e justificativas estão registrados.
- [ ] Decisão final está assinada pelo decisor.
- [ ] Condições (se Go com condições) têm prazo e responsável.
- [ ] Resultado foi comunicado a todas as partes.
- [ ] Próximos passos estão definidos (seja Go ou No-Go).
