# Cross-Squad Coordination Checklist

## Propósito
Garantir que handoffs entre squads estão completos, com contratos claros e expectativas alinhadas.

## Quando Usar
- Durante a fase correspondente do pipeline de pré-programação
- Em revisões de qualidade antes de avançar para a próxima fase
- Quando o Readiness Gatekeeper solicita verificação

---

## Checklist

### Handoff para Squads
- [ ] Artefatos de handoff estão completos para cada squad receptor
- [ ] Contratos de interface entre squads estão definidos
- [ ] Dependências cross-squad estão mapeadas e comunicadas
- [ ] SLAs de resposta entre squads estão acordados
- [ ] Pontos de contato estão identificados

### Alinhamento
- [ ] Design Squad validou requisitos de UX/UI
- [ ] Data Squad validou requisitos de dados
- [ ] Cybersecurity Squad validou requisitos de segurança
- [ ] C-Level está informado do status e riscos
- [ ] Coding Squad recebeu pacote completo para implementação

---

## Critérios de Aprovação
- **Mínimo:** 80% dos itens marcados
- **Recomendado:** 100% dos itens marcados
- **Bloqueador:** Qualquer item crítico não atendido deve ser escalado

## Red Flags
- Mais de 3 itens não marcados sem justificativa
- Itens marcados sem evidência de verificação real
- Checklist preenchido em menos de 5 minutos (superficialidade)
