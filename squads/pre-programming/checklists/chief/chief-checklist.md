# Pre-Programming Chief Checklist

## Propósito
Garantir que o Chief orquestrou corretamente o fluxo do projeto, priorizou tarefas e manteve governança.

## Quando Usar
- Durante a fase correspondente do pipeline de pré-programação
- Em revisões de qualidade antes de avançar para a próxima fase
- Quando o Readiness Gatekeeper solicita verificação

---

## Checklist

### Orquestração
- [ ] Pipeline de fases está definido e comunicado
- [ ] Agentes corretos foram acionados para cada fase
- [ ] Prioridades estão claras e documentadas
- [ ] Win condition do projeto está definida
- [ ] Cronograma de gates está estabelecido

### Governança
- [ ] Decisões críticas estão registradas no decision log
- [ ] Escalações foram tratadas dentro do SLA
- [ ] Stakeholders foram informados nos checkpoints
- [ ] Riscos de prazo foram comunicados proativamente
- [ ] Retrospectiva (RalphLoop) está agendada

---

## Critérios de Aprovação
- **Mínimo:** 80% dos itens marcados
- **Recomendado:** 100% dos itens marcados
- **Bloqueador:** Qualquer item crítico não atendido deve ser escalado

## Red Flags
- Mais de 3 itens não marcados sem justificativa
- Itens marcados sem evidência de verificação real
- Checklist preenchido em menos de 5 minutos (superficialidade)
