# Validar Suposições

## Descrição
Validar suposições identificadas durante discovery com dados e evidências concretas.

## Agentes Envolvidos
Consultar `config.yaml` para roteamento de agentes por tipo de task.

## Passos

### 1. Listar todas as suposições do assumptions log

### 2. Priorizar por risco (impacto se errada × probabilidade de estar errada)

### 3. Definir método de validação para cada suposição

### 4. Executar validação (dados, entrevistas, provas de conceito)

### 5. Atualizar assumptions log com resultado

## Artefatos de Saída
- Documento de resultado seguindo template correspondente
- Atualização dos registries relevantes
- Input para a próxima fase do pipeline

## Critérios de Conclusão
- [ ] Todos os passos foram executados
- [ ] Artefatos de saída estão completos
- [ ] Registries foram atualizados
- [ ] Checklist de qualidade correspondente está aprovado
