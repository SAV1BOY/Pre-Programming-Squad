# Workflow 02: Da Discovery ao Escopo

## Objetivo
Investigar o problema em profundidade e transformar o entendimento em uma definição de escopo clara, priorizada e aprovada.

## Trigger
- Project Brief aprovado no workflow 01.

## Agentes Envolvidos
- Agente de Discovery
- Agente de Alinhamento
- Agente de Escopo
- Agente de Risco
- Stakeholders

## Passos

### 1. Executar Problem Framing
- Executar task `run-problem-framing.md`.
- Preencher template `discovery-template.md`.
- **Output:** Declaração do problema e mapa As-Is.

### 2. Mapear Stakeholders
- Executar task `map-stakeholders.md`.
- Preencher template `stakeholder-alignment-template.md`.
- **Output:** Mapa de stakeholders com estratégia de engajamento.

### 3. Identificar Restrições
- Executar task `identify-constraints.md`.
- Preencher template `constraint-map-template.md`.
- **Output:** Mapa de restrições categorizado.

### 4. Definir Critérios de Sucesso
- Executar task `define-success.md`.
- **Output:** Critérios de sucesso mensuráveis e acordados.

### 5. Definir MVP
- Executar task `define-mvp.md`.
- **Output:** Escopo MVP definido.

### 6. Dividir em Fases
- Executar task `split-phases.md`.
- **Output:** Roadmap de fases.

### 7. Marcar Out-of-Scope
- Executar task `mark-out-of-scope.md`.
- **Output:** Lista de itens fora do escopo.

### 8. Priorizar Riscos
- Executar task `prioritize-risks.md`.
- Preencher template `risk-register-template.md`.
- **Output:** Registro de riscos priorizado.

### 9. Aprovar Escopo
- Preencher template `scope-definition-template.md`.
- Apresentar para stakeholders e obter aprovação.
- **Output:** Escopo aprovado.

## Gates de Qualidade
- [ ] O problema está enquadrado sem referência a soluções.
- [ ] Stakeholders estão mapeados com estratégia de engajamento.
- [ ] Restrições estão identificadas e classificadas.
- [ ] Critérios de sucesso são mensuráveis.
- [ ] MVP está definido e é coerente.
- [ ] Itens fora do escopo estão explícitos.
- [ ] Riscos críticos têm plano de mitigação.
- [ ] Escopo está aprovado por stakeholders.

## Output
- Discovery template completo.
- Scope definition aprovado.
- Risk register preenchido.
- Constraint map preenchido.
- Stakeholder alignment preenchido.
