# Executar Verificação de Compliance

## Descrição
Verificar se o design atende requisitos regulatórios e de compliance antes da implementação.

## Agentes Envolvidos
Consultar `config.yaml` para roteamento de agentes por tipo de task.

## Passos

### 1. Identificar regulamentações aplicáveis (LGPD, PCI-DSS, etc)

### 2. Verificar classificação de dados sensíveis

### 3. Verificar requisitos de auditoria e logging

### 4. Validar com Security & Trust Reviewer

### 5. Documentar gaps de compliance

## Artefatos de Saída
- Documento de resultado seguindo template correspondente
- Atualização dos registries relevantes
- Input para a próxima fase do pipeline

## Critérios de Conclusão
- [ ] Todos os passos foram executados
- [ ] Artefatos de saída estão completos
- [ ] Registries foram atualizados
- [ ] Checklist de qualidade correspondente está aprovado
