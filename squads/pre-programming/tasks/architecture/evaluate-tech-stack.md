# Avaliar Stack Tecnológica

## Descrição
Avaliar e recomendar stack tecnológica para o projeto com base em requisitos, constraints e contexto.

## Agentes Envolvidos
Consultar `config.yaml` para roteamento de agentes por tipo de task.

## Passos

### 1. Listar requisitos técnicos e não-funcionais

### 2. Identificar constraints (equipe, infra, budget, timeline)

### 3. Pesquisar opções viáveis de stack

### 4. Avaliar cada opção contra critérios objetivos

### 5. Documentar trade-offs e recomendação no ADR

## Artefatos de Saída
- Documento de resultado seguindo template correspondente
- Atualização dos registries relevantes
- Input para a próxima fase do pipeline

## Critérios de Conclusão
- [ ] Todos os passos foram executados
- [ ] Artefatos de saída estão completos
- [ ] Registries foram atualizados
- [ ] Checklist de qualidade correspondente está aprovado
