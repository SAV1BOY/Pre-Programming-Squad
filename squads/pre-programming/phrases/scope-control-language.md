# Frases: Controle de Escopo

## Contexto
Usar quando o escopo está se expandindo (scope creep) ou quando novos pedidos surgem sem análise de impacto. Estas frases ajudam a manter o escopo sob controle sem parecer inflexível.

## Frases-Modelo

1. "Isso é uma ótima ideia. Ela está no escopo aprovado ou é uma adição? Se é adição, qual é o impacto no prazo?"

2. "Podemos incluir isso, mas algo precisa sair. O que tiramos para abrir espaço?"

3. "Esse pedido está alinhado com o MVP ou é material para a fase 2?"

4. "Vamos anotar essa ideia no backlog de fase 2. Ela não se perde, mas também não atrasa o que já combinamos."

5. "Cada funcionalidade nova tem um custo: desenvolvimento, teste, manutenção e complexidade. O valor justifica?"

6. "O escopo foi aprovado por todos os stakeholders. Mudar agora exige re-aprovação e pode impactar o prazo em X dias."

7. "Escopo que cresce em silêncio é a principal causa de projetos atrasados. Vamos manter a disciplina de avaliar o impacto de cada adição."

8. "Sou a favor de incluir se o valor for claro. Mas preciso que o impacto no cronograma esteja explícito antes de aprovar."

## Quando Usar
- Quando novos requisitos surgem após o escopo ser aprovado.
- Quando stakeholders pedem "só mais uma coisinha".
- Quando a equipe está adicionando funcionalidades por conta própria.

## Quando NÃO Usar
- Para bloquear mudanças legítimas e necessárias.
- Quando o escopo original está genuinamente errado e precisa mudar.
- Em contextos exploratórios onde o escopo é deliberadamente fluido.

## Tom Recomendado
Disciplinado mas flexível. Não se trata de dizer "não" — se trata de dizer "sim, e aqui está o impacto". O objetivo é tornar o custo das mudanças visível para que decisões sejam informadas.
