# Frases: Mentalidade Test-First

## Contexto
Usar para promover a cultura de pensar em testes e critérios de aceite antes da implementação, garantindo que a qualidade seja projetada desde o início.

## Frases-Modelo

1. "Se não sabemos como testar, não entendemos o suficiente para implementar."

2. "Vamos definir o critério de aceite antes de escrever o código. Assim sabemos quando terminar."

3. "Como vamos provar que isso funciona? Se não temos resposta, o requisito não está pronto."

4. "Pensar no teste antes do código não é overhead — é design. Os melhores designs surgem de pensar em como verificar."

5. "Testes não são uma etapa depois do desenvolvimento. São parte do desenvolvimento."

6. "Se o requisito não é testável, ele não é um requisito — é um desejo. Vamos transformá-lo em algo verificável."

7. "Qual é o pior cenário que pode acontecer com essa funcionalidade? Temos um teste para isso?"

8. "Edge cases identificados no planejamento viram testes. Edge cases descobertos em produção viram incidentes."

## Quando Usar
- Quando requisitos estão sendo definidos sem critérios de aceite.
- Quando a equipe trata testes como "coisa para depois".
- Quando cenários de edge case não estão sendo considerados no design.

## Quando NÃO Usar
- Quando a equipe já tem cultura forte de test-first.
- Em prototipagem exploratória onde o objetivo é aprender, não entregar.
- Para exigir cobertura de testes irrealista em provas de conceito.

## Tom Recomendado
Encorajador e educativo. A mentalidade test-first é uma habilidade que se desenvolve. Não critique quem não pensa assim naturalmente — demonstre o valor através de exemplos concretos de problemas que testes antecipados teriam evitado.
