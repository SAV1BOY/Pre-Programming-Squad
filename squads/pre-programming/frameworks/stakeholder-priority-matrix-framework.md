# Stakeholder Priority Matrix Framework

## Propósito
Framework para mapear, classificar e gerenciar expectativas de stakeholders com base em influência, interesse e alinhamento.

## Quando Usar
- Durante a fase de pré-programação quando o contexto exige este tipo de análise
- Quando há complexidade ou incerteza no aspecto coberto por este framework
- Como parte do pipeline padrão para projetos de média/alta complexidade

## Processo / Passos

### Passo 1
Listar todos os stakeholders do projeto

### Passo 2
Classificar por: influência (alta/baixa) × interesse (alto/baixo)

### Passo 3
Mapear expectativas explícitas e implícitas de cada um

### Passo 4
Identificar conflitos de interesse entre stakeholders

### Passo 5
Definir estratégia de comunicação para cada quadrante

### Passo 6
Validar alinhamento em checkpoints do pipeline

## Armadilhas Comuns

- **Tratar todos os stakeholders com a mesma prioridade**
- **Ignorar stakeholders de baixa influência mas alto interesse**
- **Não identificar conflitos de interesse cedo**
- **Assumir que silêncio significa concordância**

## Output Esperado
Documento estruturado com análise, decisões e justificativas seguindo os passos acima.
